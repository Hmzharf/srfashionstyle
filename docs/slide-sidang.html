SR Fashion Style — Sidang KP02
Pendahuluan

Latar Belakang & Permasalahan

Proses penjualan, pencatatan transaksi, dan pengelolaan stok di SR Fashion Style masih dilakukan secara manual dan belum terintegrasi, sehingga rawan kesalahan dan jangkauan pemasaran terbatas.

1 Manual

Pencatatan transaksi manual sehingga rawan terjadi kesalahan.

2 Stok

Pengelolaan stok belum terorganisir & sulit dipantau real-time.

3 Offline

Belum ada penjualan online, pelanggan harus datang ke toko.

4 Pasar

Jangkauan pemasaran masih terbatas pada area sekitar.

5 Terpisah

Tidak ada integrasi antara penjualan offline dan online.

→ Solusi

Sistem Marketplace + POS terintegrasi berbasis web.

Perancangan Sistem

Arsitektur & Teknologi

Pengguna→ Frontend — React (SPA)→ Backend — Laravel REST API→ Database — MySQL

⚙ Teknologi

  • Backend: Laravel 10 (PHP 8.1) + Sanctum (auth token)
  • Frontend: React 19 + Vite + React Router
  • Database: MySQL / MariaDB
  • Pola: Client–Server terpisah (REST API / JSON)

🔗 Integrasi Pihak Ketiga

  • Midtrans — payment gateway pembayaran online
  • Biteship — cek ongkir & pelacakan resi (J&T)
  • SMTP Mail — verifikasi email OTP & notifikasi

SR Fashion Style — Sidang KP09
Perancangan Sistem

Perancangan Basis Data

Basis data dirancang dengan ERD → Transformasi → LRS, menghasilkan tabel-tabel terhubung yang menyatukan data online dan POS:

P Produk

products · variants · images · inventories · categories

O Order

orders · order_items · reviews

K POS

cashier_staff · pos_shifts · pos_transactions · items

Relasi kunci: 1 produk → banyak varian → 1 inventori; 1 order → banyak item; 1 shift kasir → banyak transaksi POS.

🗄 Ringkasan

  • 15 tabel utama (+ 2 sistem)
  • 6 modul bisnis
  • Integritas data: foreign key & cascade
  • Stok tersinkron lintas kanal
